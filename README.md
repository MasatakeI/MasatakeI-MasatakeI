
### フロントエンドの「テストのしやすさ」と「堅牢な設計」を追求するエンジニア
[![Findy Score 61](https://img.shields.io/badge/Findy_Score-61-green?logo=github&style=flat-square)](https://findy-code.io/skills-share/nR0KwIPkJyIaG)

実務未経験からエンジニアを目指し、**「保守性の高いコードとは何か」**を自問自答しながら開発に取り組んでいます。
単に「動くもの」を作るだけでなく、**テストのしやすさ**を考慮した設計（SoC: 関心の分離）を徹底することで、バグの混入を防ぎ、変化に強いアプリケーションを構築することを得意としています。

---

###  プロジェクト

#### [クイズアプリ（React / Redux Toolkit / Firebase）](https://github.com/MasatakeI/quiz_app)
**「実務レベルのテスト網羅率」と「設計の完遂」**をテーマとしたクイズアプリケーションです。

- **主なハイライト**:
  - **高いテスト品質**: Vitestを用い、主要ロジックにおいて**分岐網羅率（Branch Coverage）100%**を達成。
  - **洗練された設計**: 4層構造のアーキテクチャを採用し、UI・ロジック・状態管理・インフラを疎結合に分離。
  - **自動化されたCI/CD**: GitHub Actionsを活用し、テストからデプロイまでのパイプラインを自動化。
- **技術スタック**: React, Redux Toolkit, Firebase, Vitest, Tailwind CSS, GitHub Actions
- **リンク**: [ライブデモ](https://quiz-app-zeta-pearl.vercel.app/) / [READMEで設計詳細を見る](https://github.com/MasatakeI/quiz_app#readme)
